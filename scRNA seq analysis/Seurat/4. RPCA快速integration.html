4. RPCA快速integration

下面展示如何实现不通过规范相关分析（‘CCA’）来确定锚点, 而是使用互补主成分分析（‘RPCA’）

数据整合确定锚点有2种方法:

  • 互补主成分分析, reciprocal PCA, RPCA
    • RPCA确定任意两个数据集之间的锚点时，将每个数据集投影到另一个的PCA空间，并通过相同的互相邻近的要求约束锚点
    • 基于RPCA的整合运行速度明显更快，并更为保守的方法，如果是处于不同生物学状态的细胞在整合后可能对齐效果比较差
  • 规范相关分析, canonical correlation analysis, CCA
    • CCA通过确定数据集之间的共享变异源, 因此很适合实验之间的细胞基因表达存在非常大的差异时使用
    • 当数据集中的大部分细胞不重叠时，基于CCA的整合可能导致过度校正

适合RPCA的情况
以下情况下推荐使用RPCA：

  • 一个数据集中的大部分细胞在另一个数据集中没有匹配的类型
  • 数据集来自同一平台（例如，10x genomics的多个通道）
  • 有大量的数据集或细胞需要整合

library(SeuratData)

# load dataset
ifnb <- LoadData("ifnb")

# split the dataset into a list of two seurat objects (stim and CTRL)
ifnb.list <- SplitObject(ifnb, split.by = "stim")

# normalize and identify variable features for each dataset independently
ifnb.list <- lapply(X = ifnb.list, FUN = function(x) {
    x <- NormalizeData(x)
    x <- FindVariableFeatures(x, selection.method = "vst", nfeatures = 2000)
})

预处理
...

SelectIntegrationFeatures()函数接受一个列表, 返回一个字符串向量, 为得分最高的2000个基因名

# select features that are repeatedly variable across datasets for integration run PCA on each
# dataset using these features
features <- SelectIntegrationFeatures(object.list = ifnb.list)
ifnb.list <- lapply(X = ifnb.list, FUN = function(x) {
    x <- ScaleData(x, features = features)
    x <- RunPCA(x, features = features)
})
