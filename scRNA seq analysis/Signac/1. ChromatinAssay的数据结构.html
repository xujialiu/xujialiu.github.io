1. ChromatinAssay的数据结构

Signac包是Seurat的扩展，专为单细胞基因组测序分析设计, Signac通过向Seurat对象添加不同的测序来简单直接地分析多模态单细胞数据
这包括生成映射到基因组坐标的信号的任何测序，例如scATAC-seq、scCUT&Tag、scACT-seq和其他方法。

由于这些单细胞染色质数据集的分析相对于scRNA-seq数据的分析呈现出一些独特的挑战，我们创建了一个扩展的Assay类来存储所需的附加信息，包括：

  • Genomic ranges associated with the features (eg, peaks or genomic bins)
  • Gene annotations
  • Genome information
  • TF motifs
  • Genome-wide signal in a disk-based format (fragment files)
  • TF footprinting data
  • Tn5 insertion bias data
  • Linked genomic regions

ChromatinAssay类
...

ChromatinAssay类扩展了标准的Seurat测序类，并为单细胞染色质数据集的分析增加了一些附加的数据槽。该类包括在标准Seurat测序中的所有槽，再加上以下附加槽：

  • ranges：一个GRanges对象，包含数据矩阵中每个特征的基因组坐标
    • seqnames: 染色体号
    • ranges: 峰的位置
    • strand: 链方向（+链, -链, *链: 方向是不确定的或不相关的）
  • motifs：一个Motif对象
  • fragments：片段对象的列表
  • seqinfo：一个包含数据映射到的基因组信息的Seqinfo对象
    • seqnames
    • seqlengths
    • is_circular
    • genome
  • annotation：包含基因注释的GRanges对象
  • bias：包含Tn5整合偏见信息的向量（Tn5在不同的六聚体中的整合频率）
  • positionEnrichment：包含Tn5整合的位置富集分数的命名矩阵列表（例如，TSS或不同TF motif的富集）
  • links：描述关联的基因组位置的GRanges对象，例如共存取位点或增强子-基因调控关系。

GetAssayData和SetAssayData获取和设置ChromatinAssay数据
...

我们可以像为标准测序对象那样为ChromatinAssay获取/设置数据：使用在Seurat中定义的GetAssayData和SetAssayData函数。例如：

## Getting
data <- GetAssayData(atac_small, slot = "data")
bias <- GetAssayData(atac_small, slot = "bias")

# create a dummy bias vector
bias <- rep(1, 100)

## Setting
atac_small <- SetAssayData(atac_small, slot = "data", new.data = data)
atac_small <- SetAssayData(atac_small, slot = "bias", new.data = bias)

Fragment类
...

Fragment类用于存储和交互常用于单细胞染色质数据的片段文件。
Fragment类包含:

  • path: 片段文件的路径, 可以是http或ftp访问的远程文件
  • hash: 片段文件及其索引的MD5哈希值
  • cells: 片段文件中包含的细胞名的向量, 是一个命名向量, 允许在硬盘上的细胞名与R中的细胞名之间进行映射
    • 向量的元素: 片段文件中细胞名
    • 每个元素名字: 存储Fragment对象的ChromatinAssay对象中出现的细胞名,

向ChromatinAssay添加Fragment对象
ChromatinAssay对象可以包含一系列的Fragment对象。这样可以避免在硬盘上合并片段文件，并简化合并或整合包含ChromatinAssays的不同Seurat对象的过程。要向ChromatinAssay或包含ChromatinAssay的Seurat对象添加新的Fragment对象，我们可以使用Fragments<-赋值函数。这将执行几个操作：

重新计算片段文件和索引的MD5哈希，并验证它是否与创建Fragment对象时计算的哈希相匹配。
检查在添加的Fragment对象中包含的所有细胞是否已经包含在ChromatinAssay中存储的另一个Fragment对象中。
一个细胞的所有片段必须只存在于一个片段文件中。

将Fragment对象添加到ChromatinAssay中存储的Fragment对象列表中。

Fragments(atac_small) <- fragments

可以在创建ChromatinAssay对象时, 使用CreateChromatinAssay()函数的fragments参数传递Fragment对象或一个片段文件的路径

chrom_assay <- CreateChromatinAssay(
  counts = counts,
  genome = "hg19",
  fragments = frag.path
)
seurat.object <- CreateSeuratObject(
  counts = chrom_assay,
  assay = "peaks"
)

也可以使用SetAssayData()函数创建添加Fragment对象到ChromatinAssay对象

chrom_assay <- SetAssayData(chrom_assay, slot = "fragments", new.data = fragments)
# 或
Fragments(chrom_assay) <- fragments
# 或
chrom_assay@fragments <- fragments

从ChromatinAssay中移除一个Fragment对象

Fragments(chrom_assay) <- NULL
# 或
chrom_assay@fragments <- NULL

Motif类
...

Motif类存储DNA序列motif分析所需的信息，具有以下插槽：

  • data：一个稀疏的特征与motif矩阵，其中条目为1表示该特征包含motif，否则为0。
  • pwm：包含位置权重/位置频率矩阵的命名列表。
  • motif.names：motif ID及常见名称的列表。
  • positions：包含每个motif的确切位置的GRangesList对象
  • meta.data：关于motifs的附加信息。
    这些插槽中的都是可选的，不需要填充，但只有在运行某些函数时才需要

构建Motif类
可以使用CreateMotifObject()函数构建一个Motif对象。
构建Motif对象所需的大部分数据可以使用来自TFBSTools和motifmatchr包的函数生成。
motif的位置频率矩阵可以使用Bioconductor上的JASPAR包或chromVARmotifs包加载
